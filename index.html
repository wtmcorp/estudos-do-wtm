SEMANA 1–2

Excel Avançado Focado em Dados

Domine as ferramentas essenciais que todo analista de dados precisa

01

Tabela Dinâmica

📖 O que estudar:

  • Como transformar dados brutos em tabelas dinâmicas
  • Filtros, segmentação de dados e campos calculados
  • Agrupamento por datas (mês, trimestre, ano)
  • Atualização automática de tabelas dinâmicas

� Vídeo-aulas recomendadas:

📌 Ninja do Excel — Tabela Dinâmica Completa
📌 Hashtag Treinamentos — Tabela Dinâmica do Zero ao Avançado

🏋️ Exercício Passo a Passo:

  1. Acesse kaggle.com/datasets e baixe o dataset "Superstore Sales" (ou "Sample Superstore")
  2. Abra no Excel. Selecione todos os dados (Ctrl+T) e transforme em Tabela formatada
  3. Vá em Inserir → Tabela Dinâmica. Coloque em uma nova planilha
  4. Arraste "Data do Pedido" para LINHAS, "Categoria" para COLUNAS e "Vendas" para VALORES
  5. Clique com botão direito na data → Agrupar → Selecione "Meses" e "Anos"
  6. Adicione Segmentação de Dados: vá em Analisar → Inserir Segmentação → escolha "Região" e "Subcategoria"
  7. Crie um campo calculado: Analisar → Campos, Itens e Conjuntos → Campo Calculado → Margem = Lucro/Vendas
  8. Formate os valores como moeda (R$) e percentual (%)
💡 Dica: Após terminar, clique com botão direito na tabela → Opções → marque "Atualizar ao abrir o arquivo" para atualização automática.
02

PROCV / XLOOKUP

📖 O que estudar:

  • PROCV: sintaxe, erros comuns, limitações
  • XLOOKUP: busca bidirecional, valores padrão
  • Quando usar cada uma
  • Combinação com CORRESP e ÍNDICE

� Vídeo-aulas recomendadas:

📌 Hashtag Treinamentos — PROCV Completo (do básico ao avançado)
📌 PROCX (XLOOKUP) — A função que substitui o PROCV

🏋️ Exercício Passo a Passo:

  1. Crie uma planilha "Pedidos" com colunas: ID_Pedido, ID_Cliente, Produto, Valor, Data
  2. Crie outra planilha "Clientes" com: ID_Cliente, Nome, Cidade, Estado
  3. Preencha Pedidos com 20 registros e Clientes com 10 registros
  4. Na planilha Pedidos, use =PROCV(B2;Clientes!A:B;2;0) para trazer o nome do cliente
  5. Teste o que acontece quando o ID não existe (vai retornar #N/D)
  6. Agora use =XLOOKUP(B2;Clientes!A:A;Clientes!B:B;"Não encontrado")
  7. Compare: o XLOOKUP já trata o erro automaticamente com o 4º argumento
  8. Desafio: use XLOOKUP para buscar a Cidade do cliente pelo Nome (busca reversa — impossível com PROCV!)
💡 Dica: Em entrevistas, sempre mencione que sabe XLOOKUP. Mostra que você está atualizado. Mas saiba PROCV também, pois muitas empresas usam Excel antigo.
03

Análise de Dataset

📖 O que estudar:

  • Como explorar dados: estatísticas descritivas
  • Identificar padrões e outliers
  • Gráficos: barras, linhas, dispersão, combinados
  • Formatação condicional para destacar insights

� Vídeo-aulas recomendadas:

📌 Análise de Dados no Excel — Passo a Passo Completo
📌 Como Criar Gráficos Profissionais no Excel

🏋️ Exercício Passo a Passo:

  1. Baixe o dataset "Superstore Sales" do Kaggle e abra no Excel
  2. Use MÉDIA, MEDIANA, MÍN, MÁX para entender a distribuição de vendas
  3. Aplique Formatação Condicional (Página Inicial → Formatação Condicional → Escalas de Cor) na coluna de Lucro
  4. Crie um gráfico de barras: Top 10 produtos mais vendidos (selecione dados → Inserir → Gráfico de Barras)
  5. Crie um gráfico de linhas: Tendência de vendas mensais ao longo do ano
  6. Crie um gráfico combinado: Vendas (barras) vs Lucro (linha) por categoria
  7. Formate profissionalmente: remova linhas de grade, adicione rótulos de dados, escolha cores harmônicas
  8. Escreva 3 insights que você descobriu (ex: "Tecnologia tem maior receita mas Suprimentos tem maior volume")
💡 Dica: Recrutadores adoram ver que você sabe extrair insights dos dados, não apenas criar gráficos bonitos. Sempre documente suas conclusões.
04

Limpeza de Dados

📖 O que estudar:

  • Remover duplicatas e espaços extras (ARRUMAR/TRIM)
  • Padronizar texto (MAIÚSCULA, MINÚSCULA, PRI.MAIÚSCULA)
  • Tratar valores em branco e #N/D
  • Texto para Colunas, Flash Fill
  • Validação de dados

� Vídeo-aulas recomendadas:

📌 Limpeza e Tratamento de Dados no Excel

🏋️ Exercício Passo a Passo:

  1. Crie uma planilha "suja" com 30 registros contendo: nomes com espaços extras (" João "), CPFs sem formatação, datas em formatos diferentes (01/03/2025, 2025-03-01, 1 de março)
  2. Use =ARRUMAR(A2) para remover espaços extras dos nomes
  3. Use =PRI.MAIÚSCULA(A2) para padronizar capitalização
  4. Vá em Dados → Remover Duplicatas e elimine registros repetidos
  5. Para tratar #N/D, use =SEERRO(PROCV(...);"") para substituir erros por vazio
  6. Use Texto para Colunas (Dados → Texto para Colunas) para separar "Cidade-Estado" em duas colunas
  7. Aplique Validação de Dados (Dados → Validação) para criar lista suspensa de estados válidos
  8. Ao final, compare o antes e depois: conte registros limpos vs originais
💡 Dica: Na vida real, 80% do trabalho de um analista é limpeza de dados. Quem domina isso se destaca em qualquer vaga.
SEMANA 3–4

Power BI

Transforme dados em dashboards interativos e impressionantes

01

Fundamentos e Interface

📖 O que estudar:

  • Interface do Power BI Desktop (3 visões)
  • Power Query: importar e transformar dados
  • Tipos de dados e formatação
  • Publicação no Power BI Service

🎬 Vídeo-aulas recomendadas:

📌 Hashtag Treinamentos — Curso Básico Power BI (do Zero)
📌 Aprenda Power BI do Zero em Uma Aula

🏋️ Exercício Passo a Passo:

  1. Baixe e instale o Power BI Desktop (gratuito) em powerbi.microsoft.com
  2. Abra o Power BI e explore as 3 visões: Relatório, Dados e Modelo
  3. Clique em Obter Dados → Excel e importe o dataset Superstore que você usou no Excel
  4. No Power Query, renomeie colunas, altere tipos de dados e remova colunas desnecessárias
  5. Clique em Fechar e Aplicar para carregar os dados
  6. Crie um visual simples: arraste "Vendas" para um Cartão e "Categoria" para um gráfico de barras
💡 Dica: O Power Query é o "coração" do Power BI. Domine ele e você resolve 50% dos problemas antes de começar o dashboard.
02

Modelagem de Dados

📖 O que estudar:

  • Tabelas fato vs tabelas dimensão
  • Relacionamentos 1:N e N:N
  • Tabela calendário (dCalendario)
  • Boas práticas de modelagem

🎬 Vídeo-aula recomendada:

📌 Star Schema (Modelo Estrela) no Power BI — Explicação Completa

🏋️ Exercício Passo a Passo:

  1. Crie 3 planilhas no Excel: fVendas (ID_Venda, ID_Produto, ID_Loja, Data, Valor), dProduto (ID_Produto, Nome, Categoria), dLoja (ID_Loja, Cidade, Estado)
  2. Importe todas no Power BI via Obter Dados → Excel
  3. Vá para a visão Modelo e crie relacionamentos: fVendas[ID_Produto] → dProduto[ID_Produto]
  4. Crie uma tabela calendário em DAX: dCalendario = CALENDAR(DATE(2024,1,1), DATE(2025,12,31))
  5. Relacione fVendas[Data] com dCalendario[Date]
  6. Verifique se há estrela: fVendas no centro conectada a 3 dimensões ao redor
💡 Dica: Sempre nomeie tabelas fato com "f" e dimensão com "d" (ex: fVendas, dProduto). Isso mostra organização profissional.
03

Dashboards

📖 O que estudar:

  • Visuais: cartões, gráficos, mapas, tabelas
  • Filtros e segmentação de dados (Slicers)
  • Drill-down e drill-through
  • Design: cores, alinhamento, hierarquia visual
  • Bookmarks e navegação entre páginas

🏋️ Exercício Passo a Passo:

  1. Com os dados já importados, crie uma página "Visão Geral" no Power BI
  2. Adicione 4 Cartões KPI: Total Vendas, Total Lucro, Qtd Pedidos, Ticket Médio
  3. Crie um gráfico de barras com Vendas por Categoria
  4. Crie um gráfico de linhas com Vendas por Mês (use a tabela calendário)
  5. Adicione Slicers (segmentadores) de Ano, Região e Categoria
  6. Aplique um tema profissional: Exibir → Temas → escolha um tema escuro
  7. Alinhe todos os visuais usando Formatar → Alinhar e distribua uniformemente
  8. Teste a interatividade: clique em uma barra e veja os outros visuais filtrarem automaticamente
💡 Dica: Um dashboard bonito com 5 visuais bem feitos impressiona mais que 15 visuais desorganizados. Menos é mais!
04

DAX e Métricas

📖 O que estudar:

  • Medidas vs Colunas Calculadas
  • SUM, AVERAGE, COUNT, DISTINCTCOUNT
  • CALCULATE, FILTER, ALL
  • Inteligência temporal: TOTALYTD, SAMEPERIODLASTYEAR
  • FORMAT e funções de texto

🎬 Vídeo-aula recomendada:

📌 O que é DAX Power BI? (Sem Complicação)

🏋️ Exercício Passo a Passo — Crie estas medidas DAX:

Total Vendas = SUM(fVendas[Valor]) Ticket Médio = AVERAGE(fVendas[Valor]) Qtd Pedidos = COUNTROWS(fVendas) Vendas YTD = TOTALYTD([Total Vendas]; dCalendario[Date]) Vendas Ano Anterior = CALCULATE([Total Vendas]; SAMEPERIODLASTYEAR(dCalendario[Date])) Variação % = DIVIDE([Total Vendas] - [Vendas Ano Anterior]; [Vendas Ano Anterior])
  1. Vá em Modelagem → Nova Medida e crie cada uma das medidas acima
  2. Coloque "Total Vendas" em um Cartão e formate como moeda
  3. Crie um Cartão com "Variação %" e use formatação condicional (verde se positivo, vermelho se negativo)
  4. Teste: filtre por ano e veja se as medidas YTD e Ano Anterior mudam corretamente
💡 Dica: Sempre use DIVIDE() em vez de barra (/) no DAX. Ele trata divisão por zero automaticamente.

01

SELECT e Filtros

📖 O que estudar:

  • SELECT *, SELECT colunas específicas
  • WHERE com operadores: =, !=, >, <, BETWEEN, IN, LIKE
  • ORDER BY ASC/DESC
  • LIMIT e OFFSET
  • Alias com AS

� Vídeo-aulas recomendadas:

📌 Hashtag Treinamentos — SQL do Zero em menos de 15 min
📌 Curso Completo SQL para Iniciantes

🏋️ Exercício Passo a Passo:

  1. Acesse o SQLite Online
  2. Crie uma tabela: CREATE TABLE vendas (id INT, produto TEXT, valor REAL);
  3. Insira 5 produtos com valores diferentes usando INSERT INTO
  4. Faça um SELECT * para ver todos os dados
  5. Filtre produtos com valor acima de 50: WHERE valor > 50
  6. Ordene do mais caro para o mais barato: ORDER BY valor DESC
💡 Dica: O SQL é a linguagem universal dos dados. Se você souber SQL, consegue trabalhar com qualquer banco de dados do mercado.
02

JOIN

📖 O que estudar:

  • INNER JOIN: combinar tabelas pela chave
  • LEFT JOIN: manter todos da tabela esquerda
  • RIGHT JOIN e FULL OUTER JOIN
  • JOIN com múltiplas condições

� Vídeo-aulas recomendadas:

📌 Como funcionam os JOINs no SQL (Visual)

🏋️ Exercício Passo a Passo:

  1. No SQLite Online, crie a tabela clientes (id INT, nome TEXT)
  2. Insira clientes e relacione os IDs com a tabela de vendas que você criou antes
  3. Use INNER JOIN para listar o nome do cliente ao lado do produto que ele comprou
  4. Experimente o LEFT JOIN para ver se existem clientes que ainda não compraram nada
💡 Dica: Em 90% das vezes, você usará INNER JOIN ou LEFT JOIN. Foque em dominar esses dois primeiro.
03

GROUP BY e Agregações

📖 O que estudar:

🎬 Vídeo-aulas recomendadas:

📌 Agrupamento de Dados e Funções de Agregação

🏋️ Exercício Passo a Passo:

  1. Use SUM(valor) para ver o total vendido
  2. Use GROUP BY produto para ver quanto foi vendido de cada item
  3. Use COUNT(*) para saber quantos pedidos foram feitos
  4. Use HAVING SUM(valor) > 100 para mostrar apenas produtos que venderam mais de 100 no total
04

Consultas Avançadas

📖 O que estudar:

🎬 Vídeo-aulas recomendadas:

📌 Subqueries, CASE WHEN e Funções avançadas

🏋️ Exercício Passo a Passo:

  1. Crie uma categoria de preço usando CASE WHEN valor > 100 THEN 'Caro' ELSE 'Barato' END
  2. Use strftime('%m', data) (se estiver no SQLite) para extrair o mês da venda
  3. Tente criar uma consulta que mostre a média de vendas por mês
💡 Dica: O CASE WHEN é o "SE" do SQL. Essencial para criar faixas etárias, categorias de produtos e status.
